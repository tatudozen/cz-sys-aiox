## Dev Notes

### Deploy Pattern
[Source: architecture/copyzen-architecture.md#2.5, PRD#4.1]
```
GitHub Push → Actions: build + rsync → /srv/sites/{domain}/ na KVM4
                                     → nginx:alpine serve os arquivos
                                     → Traefik roteia + SSL
Stack Docker gerenciado via Portainer UI — NOT via Actions (NFR-15)
```

### CRÍTICO: Portainer vs Actions
[Source: PRD#NFR-15]
GitHub Actions deve APENAS fazer build + rsync de arquivos estáticos. O gerenciamento de stacks Docker (nginx, Traefik) é EXCLUSIVAMENTE via Portainer UI. Actions NUNCA deve fazer `docker stack deploy` ou gerenciar containers.

### Node Version Discrepancy
Architecture especifica Node 20 LTS; CI atual usa 22. Manter 22 (compatível e mais recente). Documentar para revisão futura.
