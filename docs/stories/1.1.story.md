### Tech Stack Reference

| Tecnologia | Versão | Uso nesta Story |
|-----------|--------|----------------|
| TypeScript | 5.x | Strict mode em todos os packages |

| Tailwind CSS | 4.x | Configurado no Astro (novo modo v4 via Vite plugin) |

*Nota: o `ci.yml` existente usa Node.js 22. A arquitetura especifica 20 LTS. Manter 22 no CI (compatível e mais recente), documentar a discrepância. [Note: fallback — architecture spec = 20 LTS, CI atual = 22]

### BrandConfig Interface
[Source: architecture/copyzen-architecture.md#4.2]

A interface `BrandConfig` definida em `packages/core` é o contrato central usado por todos os agentes e pela geração de páginas. Os campos expandidos (vs PRD) incluem `background_color`, `text_color`, `custom_guidelines`, `slogan`, `keywords`. Esta é a versão definitiva da arquitetura — implementar completa desde já para evitar breaking changes futuros.

### Tailwind CSS 4.x — Observação Importante
[Source: architecture/copyzen-architecture.md#3]

Tailwind v4 mudou o método de integração com Astro. Em v4:
- Não usa `tailwind.config.js` da mesma forma
- Usa `@astrojs/tailwind` com configuração via plugin Vite
- Verificar documentação atual de integração Astro + Tailwind v4 antes de implementar

### Testing Strategy
[Source: PRD#4.5, architecture/copyzen-architecture.md#3]

- Framework: Vitest 3.x (alinhado com Astro ecosystem)
- Esta story requer apenas 1 smoke test no `core` package
- Sem E2E no MVP — Lighthouse CI como proxy de qualidade
- Testes mais complexos (integração Supabase, agentes) chegam nas Stories 1.2 e 3.x
