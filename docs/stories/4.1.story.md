# Story 4.1: Content Generation Pipeline (n8n Workflow)

## Story

**As a** operator (Fernando),
**I want** an n8n workflow that orchestrates content generation for a client project,
**so that** the CMO can trigger post creation end-to-end.

---

## Status

Approved

---

## Context

**Epic 4 — Sistema 1: Geração de Conteúdo (Inception & Atração Fatal)**

Dependência de Epic 3 (agentes Copywriter e Designer funcionais). Orquestra o pipeline de conteúdo completo: CMO define plano → Copywriter gera copy → Designer gera prompts de imagem → Image API gera imagens → salva outputs.

---

## Acceptance Criteria

- [ ] **AC-1:** n8n workflow `content-generation-pipeline` com stages:
  1. Load client brief + brand config (Supabase)
  2. CMO define content plan (via agent server)
  3. Copywriter gera copy por post
  4. Designer gera prompts de imagem
  5. Image API gera imagens (Story 4.3)
  6. Package output
  7. Deliver to Google Drive (Story 4.4)

- [ ] **AC-2:** Input do workflow: `client_id`, `project_plan_id`, `mode` (inception/atracao_fatal), `quantity` (número de posts)

- [ ] **AC-3:** CMO gera content plan como JSON: array de posts com `type`, `topic`, `mode`, `cta_target` (se Atração Fatal)

- [ ] **AC-4:** Cada post passa por: Copywriter (copy) → Designer (prompt) → Image API → composição final

- [ ] **AC-5:** Output salvo em `content_outputs` (verificar se tabela existe nas migrations):
  - `id`, `project_plan_id`, `post_index`, `type`, `mode`, `copy` (jsonb), `image_prompt`, `image_url`, `status`, `created_at`

- [ ] **AC-6:** Workflow handleia falhas por post (um post com erro não cancela os demais — try/catch por iteração)

- [ ] **AC-7:** Teste: trigger com 3 posts → todos passam pelo pipeline → outputs salvos no Supabase
