# Story 2.4: Proposta & Ciclo de Aprovação

## Story

**As a** operator (Fernando),
**I want** to review, adjust if needed, and send the proposal to the client for approval,
**so that** the project only proceeds with explicit client consent.

---

## Status

Approved

---

## Context

**Epic 2 — Sistema 0: Geração de Projeto (Briefing Pipeline)**

Dependência de 2.3 (proposta gerada pelo CMO). Implementa o ciclo de revisão + envio da proposta ao cliente via Email e/ou WhatsApp, e o mecanismo de aprovação/rejeição.

---

## Acceptance Criteria

- [ ] **AC-1:** n8n workflow `send-proposal` que Fernando triggera manualmente após revisar a proposta:
  - Input: `proposal_id`
  - Carrega proposta do Supabase
  - Fernando pode ter editado os campos diretamente no Supabase antes de triggerar

- [ ] **AC-2:** Proposta enviada ao cliente via:
  - Email (n8n Email node ou GoHighLevel email automation): assunto "Proposta CopyZen — {business_name}", corpo com resumo do pacote + timeline + preço + próximos passos + link de aprovação
  - WhatsApp (Evolution API): mensagem resumida com link de aprovação

- [ ] **AC-3:** Link de aprovação: n8n webhook endpoint que registra a decisão:
  - `POST /webhook/proposal-response` com payload: `{ proposal_id, decision: "approved"|"rejected" }`

- [ ] **AC-4:** Quando `approved`:
  - Status da proposta → `approved`
  - Status do briefing → `approved`
  - Trigger automático do workflow `dispatch-specific-surveys` (Story 2.5)
  - Fernando notificado

- [ ] **AC-5:** Quando `rejected`:
  - Status da proposta → `rejected`
  - Fernando notificado com mensagem "Cliente {name} rejeitou a proposta — follow-up manual necessário"

- [ ] **AC-6:** Confirmação enviada ao cliente após aprovação: "Ótimo! Vamos começar. Em breve você receberá os próximos passos."

- [ ] **AC-7:** Teste: proposta enviada → cliente clica "aprovar" → status atualiza → trigger da Story 2.5 disparado

## Dev Notes

### Proposal Approval Flow
[Source: architecture/copyzen-architecture.md, PRD#2.4]
```
Fernando revisa proposta no Supabase (edita se necessário)
  → Triggera workflow send-proposal
  → Cliente recebe email + WhatsApp com link
  → Cliente clica → webhook n8n recebe decision
  → Approved: brief status=approved → trigger dispatch-specific-surveys
  → Rejected: Fernando notificado para follow-up manual
```

### Template de Email da Proposta
Conteúdo mínimo:
- Headline: "Proposta CopyZen para {business_name}"
- Pacote recomendado + resumo do que inclui
- Timeline estimada
- Faixa de preço
- Link CTA: "APROVAR PROPOSTA" (verde) / "SOLICITAR AJUSTES" (cinza)
- Tom: conversacional, profissional (tom da CopyZen)
