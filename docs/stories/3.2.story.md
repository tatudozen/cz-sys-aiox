# Story 3.2: Brand Config System

## Story

**As a** developer (Fernando),
**I want** a brand configuration system that loads per-client visual and voice guidelines,
**so that** all agents produce output consistent with each client's brand.

---

## Status

Approved

---

## Context

**Epic 3 — Núcleo de Agentes (CMO, Designer, Copywriter)**

Dependência de 1.2 (Supabase com tabela brand_configs), 3.1 (LLM provider), Epic 2 (brand_intelligence disponível). O Brand Config System é o contrato central entre o banco de dados e os agentes — todos os agentes o consomem para gerar output brand-consistent.

---

## Acceptance Criteria

- [ ] **AC-1:** `packages/core/src/brand/` com tipo `BrandConfig` expandido (já definido em Story 1.1 — verificar e complementar):
  - Todos os campos de cor (primary, secondary, accent, background, text)
  - Tipografia (heading_font, body_font)
  - Tom de voz + custom_guidelines
  - slogan, keywords[], reference_images[]

- [ ] **AC-2:** `BrandConfigLoader` em `packages/core/src/brand/brand-config-loader.ts`:
  - Busca config do Supabase por `client_id`
  - Cache em memória com TTL de 5 minutos
  - Consulta `brand_intelligence` para enriquecer config quando disponível (Story 2.6)
  - Retorno: `BrandConfig` completo com dados enriquecidos

- [ ] **AC-3:** `BrandGuardrails` em `packages/core/src/brand/brand-guardrails.ts`:
  - Input: `BrandConfig`
  - Output (a): system prompt additions para agentes de texto (tom de voz, keywords, guidelines)
  - Output (b): style constraints para agentes visuais (cores hex, fontes, estilo)
  - Output (c): validation checklist para pós-geração

- [ ] **AC-4:** Função `validateBrandCompliance(output: string, brandConfig: BrandConfig): ValidationResult`:
  - Verifica se output usa keywords da brand
  - Verifica tom de voz (heurística básica)
  - Retorna `{ compliant: boolean, issues: string[] }`

- [ ] **AC-5:** Seed data: brand config da CopyZen (cliente zero) inserido no Supabase como migration de seed

- [ ] **AC-6:** Testes em `packages/core/src/brand/__tests__/`:
  - Carregar config (mock Supabase) → gerar guardrails → validar output compliant → rejeitar output non-compliant

## Dev Notes

### BrandConfig Completo
[Source: architecture/copyzen-architecture.md#4.2]
```typescript
interface BrandConfig {
  id: string;
  client_id: string;
  primary_color: string;
  secondary_color: string;
  accent_color: string;
  background_color: string;
  text_color: string;
  heading_font: string;
  body_font: string;
  tone_of_voice: 'formal' | 'casual' | 'technical';
  custom_guidelines: string | null;
  logo_url: string | null;
  slogan: string | null;
  keywords: string[];
  reference_images: string[];
}
```

### Enriquecimento com BrandIntelligence
[Source: PRD#2.6-AC9]
O `BrandConfigLoader.load(clientId)` deve:
1. Buscar `brand_configs` por client_id
2. Se existir `brand_intelligence` com status='complete' para o cliente, enriquecer campos vazios/null com dados de intelligence
3. Dados de intelligence têm prioridade MENOR que dados manuais (se campo preenchido, não sobrescrever)

### BrandGuardrails System Prompt Example
```
Você está gerando conteúdo para {business_name}.
Tom de voz: {tone_of_voice} (ex: "Use linguagem casual, próxima, tuteia o leitor")
Keywords obrigatórias: {keywords.join(', ')}
Slogan da marca: {slogan}
Diretrizes customizadas: {custom_guidelines}
```
