# Story 5.1: FunWheel Base Templates & Brand Theming System

## Story

**As a** developer (Fernando),
**I want** Astro page templates that can be themed dynamically per client's brand config,
**so that** each client gets a unique-looking FunWheel without manual design work.

---

## Status

Approved

---

## Context

**Epic 5 — Sistema 2: FunWheel (A-R-T)**

Dependência de 3.2 (brand config system), 1.1 (monorepo com packages/funwheel Astro). Cria a infraestrutura de templates temáveis que todas as outras stories do Epic 5 usarão.

---

## Acceptance Criteria

- [ ] **AC-1:** `packages/funwheel/src/templates/` com templates Astro base:
  - `apresentacao.astro`, `retencao.astro`, `transformacao.astro`, `thankyou.astro`
  - Cada template usa CSS custom properties (`var(--brand-primary)`) para cores e fontes

- [ ] **AC-2:** Build-time theming: script que recebe `BrandConfig` e gera CSS custom properties injetadas no `<head>` da página

- [ ] **AC-3:** Layout components compartilhados em `packages/funwheel/src/components/`:
  - `Hero.astro`, `Section.astro`, `CTA.astro`, `LeadForm.astro`, `Footer.astro`
  - Todos brand-aware via CSS custom properties

- [ ] **AC-4:** Templates mobile-first com breakpoints: < 768px (base), 768-1024px, > 1024px

- [ ] **AC-5:** Build gera output estático por cliente: `dist/{client_slug}/apresentacao/index.html`, etc.

- [ ] **AC-6:** Lighthouse > 90 para templates com conteúdo placeholder em todos os 4 critérios

- [ ] **AC-7:** Astro build completo sem erros: `npm run build` em `packages/funwheel/`

- [ ] **AC-8:** Teste: build com 2 brand configs diferentes → output visual distinto → ambos passam Lighthouse
