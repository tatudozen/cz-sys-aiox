# Story 2.2: WhatsApp Entry Channel (Evolution API)

## Story

**As a** potential client,
**I want** to receive the survey link via WhatsApp,
**so that** I can start my briefing from my preferred channel.

---

## Status

Approved

---

## Context

**Epic 2 — Sistema 0: Geração de Projeto (Briefing Pipeline)**

Dependência de 2.1 (survey GoHighLevel configurado). A Evolution API já está rodando na KVM4 — este story integra o canal WhatsApp como entrada alternativa ao survey.

**Infraestrutura existente:**
- Evolution API rodando na KVM4 no Docker Swarm
- n8n com HTTP Request nodes disponíveis
- WhatsApp conectado via Evolution API (número da CopyZen)

## Dev Notes

### Evolution API Integration Pattern
[Source: architecture/copyzen-architecture.md#2.2]
```
WhatsApp → Evolution API (KVM4) → webhook → n8n
n8n → HTTP Request → Evolution API → WhatsApp response
```

Evolution API endpoint para envio:
```
POST http://evolution-api:8080/message/sendText/{instance_name}
Headers: apikey: {EVOLUTION_API_KEY}
Body: { "number": "5511999999999@s.whatsapp.net", "text": "..." }
```

### Env Vars Necessárias
```
EVOLUTION_API_URL=http://evolution-api:8080
EVOLUTION_API_KEY=
EVOLUTION_INSTANCE_NAME=
GHL_SURVEY_URL=  # Link do survey GoHighLevel
```
