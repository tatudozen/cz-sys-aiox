# Story 2.3: Agente CMO — Análise de Briefing & Recomendação de Pacote

## Story

**As a** operator (Fernando),
**I want** the CMO agent to analyze incoming briefings and recommend a package,
**so that** clients receive a tailored proposal based on their needs.

---

## Status

Ready for Review

---

## Context

**Epic 2 — Sistema 0: Geração de Projeto (Briefing Pipeline)**

Dependência de 2.1 (briefings chegando no Supabase). Esta story implementa a primeira chamada real ao Claude API como parte do pipeline. A versão completa do CMO Agent vem no Epic 3 (Story 3.3) — aqui implementamos o suficiente para análise de briefing e recomendação de pacote.

**Pacotes disponíveis (do PRD):**
- IA: Geração de conteúdo (posts Instagram/Facebook)
- A-R-T: FunWheel (Apresentação + Retenção + Transformação)
- E: Página de Vendas
- Combo Leads: IA + A-R-T
- Combo Cash: IA + A-R-T + E (full stack)

---

## Acceptance Criteria

- [ ] **AC-1:** n8n workflow `cmo-analyze-briefing` triggered quando briefing muda para status `received` (Supabase trigger via webhook ou polling)

- [ ] **AC-2:** Workflow chama Claude API (HTTP Request node) com prompt do CMO incluindo:
  - Briefing formatado do cliente
  - Lista dos 5 pacotes disponíveis com critérios de recomendação
  - Instrução para output JSON estruturado

- [ ] **AC-3:** Output do CMO é JSON estruturado com:
  - `recommended_package` (enum: ia/art/e/combo_leads/combo_cash)
  - `reasoning` (texto em PT-BR justificando a recomendação)
  - `estimated_timeline` (ex: "4-6 semanas")
  - `estimated_cost_range` (ex: "R$2.500-3.500")
  - `priority_systems` (array ordenado dos sistemas a implementar)

- [ ] **AC-4:** Output salvo na tabela `proposals`:
  - `id` (uuid, PK)
  - `briefing_id` (FK → briefings.id)
  - `client_id` (uuid, nullable)
  - `package` (text)
  - `reasoning` (text)
  - `timeline` (text)
  - `cost_range` (text)
  - `status` (text, CHECK: 'draft','sent','approved','rejected', default 'draft')
  - `created_at` (timestamptz)

- [ ] **AC-5:** Guardrail do CMO: se briefing tem dados insuficientes para análise → status muda para `needs_info` → Fernando é notificado

- [ ] **AC-6:** Tempo de processamento do agente < 60 segundos (NFR-03)

- [ ] **AC-7:** Teste: briefing completo → CMO analisa → proposta gerada com pacote recomendado e justificativa coerente em PT-BR

## Dev Notes

### Claude API Call Pattern
[Source: architecture/copyzen-architecture.md#3, #5.1]
```
POST https://api.anthropic.com/v1/messages
Headers:
  x-api-key: {CLAUDE_API_KEY}
  anthropic-version: 2023-06-01
  content-type: application/json
Body:
  model: claude-sonnet-4-6
  max_tokens: 1024
  messages: [{ role: "user", content: "..." }]
  system: "..."
```

### Pacotes CopyZen (para prompt do CMO)
[Source: PRD#FR-03]
```
IA: Posts para redes sociais (carrossel + imagem). Para quem precisa de presença digital.
A-R-T: Funil FunWheel (3 páginas: Apresentação, Retenção, Transformação). Para captura de leads.
E: Página de vendas long-form. Para conversão direta.
Combo Leads: IA + A-R-T. Para crescimento de audiência + captura de leads.
Combo Cash: IA + A-R-T + E. Funil completo de marketing.
```

### Nota: CMO Partial vs Full
Esta story implementa apenas `analyzeBriefing()`. A classe `CMOAgent` completa (com `createProjectPlan`, `orchestrateExecution`, `reviewOutput`) vem na Story 3.3. Aqui o CMO opera via n8n HTTP Request direto para Claude API — sem o agent server do Epic 3 ainda.
