# Story 4.4: Google Drive Delivery Integration

## Story

**As a** operator (Fernando),
**I want** finished content packages delivered to the client's shared Google Drive folder,
**so that** clients can access and approve their posts in a familiar platform.

---

## Status

Ready for Review

---

## Context

**Epic 4 — Sistema 1: Geração de Conteúdo**

Dependência de 4.1 (pipeline completo), 4.3 (imagens geradas). Implementa entrega final dos assets ao cliente via Google Drive compartilhado.

---

## Acceptance Criteria

- [x] **AC-1:** `packages/integrations/src/google-drive/` com `GoogleDriveClient` (expandindo o básico da Story 3.5):
  - `uploadFile(folderId, fileName, content, mimeType)`: upload de arquivo
  - `createFolder(parentId, name)`: criar subpasta
  - `shareWithEmail(fileId, email, role: 'reader'|'writer')`: compartilhar com cliente
  - `listFiles(folderId)`: listar arquivos

- [x] **AC-2:** Workflow de entrega no n8n `deliver-content`:
  - Cria estrutura: `CopyZen/{client_name}/{YYYY-MM-DD}/`
  - Subpastas: `carrosseis/`, `imagens/`
  - Upload de cada asset (imagens em alta resolução)
  - Upload de arquivo texto com copy/legendas/hashtags por post
  - Compartilha pasta com email do cliente

- [x] **AC-3:** Cada post entregue com:
  - Imagem(ns) em alta resolução
  - Arquivo `.txt` ou `.md` com: headline, body copy, CTA, hashtags

- [x] **AC-4:** Service account Google com credenciais via env var `GOOGLE_SERVICE_ACCOUNT_JSON` (não credenciais pessoais)

- [x] **AC-5:** Registro em tabela `deliveries` no Supabase:
  - `id`, `project_plan_id`, `client_id`, `drive_folder_url`, `files_count`, `delivered_at`

- [x] **AC-6:** Teste: content package mock → upload para Drive → verificar estrutura de pastas e arquivos
