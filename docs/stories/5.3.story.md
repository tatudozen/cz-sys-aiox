# Story 5.3: Página de Retenção (R) & Lead Magnet PDF

## Story

**As a** lead (visitor),
**I want** to download a valuable resource in exchange for my contact info,
**so that** I get immediate value while the professional captures my lead.

---

## Status

Approved

---

## Context

**Epic 5 — Sistema 2: FunWheel (A-R-T)**

Dependência de 5.1 (templates), 5.2 (CTA da Apresentação aponta aqui). Captura leads via formulário + entrega lead magnet PDF. Integra com GoHighLevel CRM.

---

## Acceptance Criteria

- [ ] **AC-1:** Template `retencao.astro` com:
  - Headline de oferta (o que o lead recebe)
  - Descrição do lead magnet (PDF)
  - Formulário com max 3 campos: nome + email ou WhatsApp
  - Botão CTA "Quero meu acesso"

- [ ] **AC-2:** Formulário faz `fetch` para GoHighLevel API criando/atualizando contato no CRM

- [ ] **AC-3:** Após submit bem-sucedido:
  - Redireciona para `thankyou.astro`
  - GoHighLevel automation envia PDF por email

- [ ] **AC-4:** PDF Generation: `packages/integrations/src/pdf-generator/`:
  - Converte conteúdo da Apresentação em PDF estilizado com brand config
  - Armazena no Supabase Storage
  - URL compartilhada via GoHighLevel automation

- [ ] **AC-5:** Lead salvo na tabela `leads` (verificar migration):
  - `id`, `client_id`, `funwheel_id`, `name`, `email`, `phone`, `source_page` (retencao), `referral_code` (nullable), `vip_access` (default false), `referral_count` (default 0), `created_at`

- [ ] **AC-6:** `referral_code`: 6 chars alphanumeric, gerado na captura, único por lead

- [ ] **AC-7:** Teste end-to-end: visitar Retenção → preencher form → lead criado no GoHighLevel + Supabase → PDF enviado por email

## Dev Notes

### Lead Entity
[Source: architecture/copyzen-architecture.md#4.2]
```typescript
interface Lead {
  id: string;
  client_id: string;
  funwheel_id: string | null;
  name: string;
  email: string | null;
  phone: string | null;
  source_page: 'apresentacao' | 'retencao' | 'transformacao' | 'sales_page';
  referral_code: string;   // 6 chars alphanumeric
  referral_count: number;
  vip_access: boolean;
  created_at: string;
}
```

### GHL Client-Side Integration
[Source: PRD#5.3-AC2, FR-15]
O formulário usa Astro Island (Preact ou React) para submit client-side:
```javascript
// Island component
const handleSubmit = async (data) => {
  await fetch('/api/capture-lead', { method: 'POST', body: JSON.stringify(data) });
  // Ou diretamente para GHL API
};
```
Preferir API route Astro → GHL para não expor GHL API key no cliente.

### PDF Generation
[Source: PRD#5.3-AC4]
Opções: puppeteer (mais fiel ao HTML), pdfkit (mais leve). Para MVP: pdfkit com template simples. Conteúdo: resumo do que foi prometido na Apresentação, lead magnet content, branding da marca do cliente.
