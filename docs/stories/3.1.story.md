# Story 3.1: LLM Abstraction Layer & Claude API Client

## Story

**As a** developer (Fernando),
**I want** an abstracted LLM interface with Claude API as primary provider,
**so that** agents can call LLMs consistently and we can add fallback providers later (NFR-10).

---

## Status

Ready for Review

---

## Context

**Epic 3 — Núcleo de Agentes (CMO, Designer, Copywriter)**

Dependência de Epic 1 (monorepo, `packages/agents/` scaffolded). Fundação do agent server — todos os agentes do Epic 3 dependem desta abstração. Implementa o `LLMProvider` interface e `ClaudeProvider` com retry logic.

- [ ] **AC-1:** `packages/agents/src/llm/` com:
  - Interface `LLMProvider`: `complete(prompt: string, options?: LLMOptions): Promise<string>`
  - Interface `LLMOptions`: `model?`, `maxTokens?`, `temperature?`, `systemPrompt?`

- [ ] **AC-4:** Configuração via env vars:
  - `LLM_PROVIDER=claude`
  - `CLAUDE_API_KEY`

- [ ] **AC-6:** Logging de cada chamada: salvar em tabela `llm_usage_log` no Supabase:
  - `id`, `provider`, `model`, `input_tokens`, `output_tokens`, `total_cost` (USD estimado), `timestamp`

  - Mock do provider → teste de retry (3 falhas → erro)
  - Teste de timeout
  - Teste de factory
  - Fallback graceful quando provider indisponível

- [ ] **AC-8:** Teste de integração: chamada real à Claude API retorna resposta válida

## Dev Notes

### Claude API — Modelo Correto
[Source: architecture/copyzen-architecture.md#3]
Usar `claude-sonnet-4-6` (não claude-3-sonnet, não claude-opus). Confirmar ID exato na documentação Anthropic antes de hardcodar. Configurável via env var `CLAUDE_MODEL`.

### LLMProvider Interface Pattern
[Source: architecture/copyzen-architecture.md#2.5 — Provider Abstraction]
```typescript
interface LLMProvider {
  complete(prompt: string, options?: LLMOptions): Promise<LLMResponse>;
  stream?(prompt: string, options?: LLMOptions): AsyncIterable<string>;
}
```
O método `stream` é opcional no MVP (implementar `complete` apenas). Streams podem ser adicionados na Story 3.3+.

### Custo Estimado Claude API
[Source: PRD#NFR-08]
Modelo sonnet: ~$3/1M input tokens, ~$15/1M output tokens (aproximado). O `total_cost` em `llm_usage_log` é estimativa para tracking de budget ($20-50/mês limite).

### LLMUsageLog Entity
[Source: architecture/copyzen-architecture.md#4.2]
```typescript
interface LLMUsageLog {
  id: string;
  provider: string;
  model: string;
  input_tokens: number;
  output_tokens: number;
  total_cost: number; // USD estimado
  timestamp: string;
}
```
