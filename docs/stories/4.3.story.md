# Story 4.3: Image Subsystem & AI Image Generation

## Story

**As a** developer (Fernando),
**I want** integration with Nano Banana and Kling AI for image generation,
**so that** the Designer agent can produce brand-consistent images automatically.

---

## Status

Ready for Review

---

## Context

**Epic 4 — Sistema 1: Geração de Conteúdo**

Dependência de 3.5 (DesignerAgent com `generateImagePrompt`). Implementa os providers de geração de imagem com fallback automático.

---

## Acceptance Criteria

- [x] **AC-1:** `packages/integrations/src/image-generation/` com:
  - Interface `ImageGenerator`: `generate(prompt: string, options: ImageOptions): Promise<ImageResult>`
  - `NanoBananaProvider` implementando `ImageGenerator`
  - `KlingAIProvider` implementando `ImageGenerator`

- [x] **AC-2:** Factory `createImageGenerator(provider?: 'nano-banana'|'kling-ai')`:
  - Fallback: se Nano Banana falhar → tenta Kling AI
  - Configurável via env var `IMAGE_PROVIDER=nano-banana`

- [x] **AC-3:** Input: prompt (string do Designer), aspect_ratio ('1:1'|'4:5'|'16:9'), style_preset derivado do brand config

- [x] **AC-4:** Output:
  ```typescript
  interface ImageResult {
    provider_url: string;   // URL retornada pelo provider
    storage_url: string;    // URL após download para Supabase Storage
    width: number;
    height: number;
    generation_id: string;
  }
  ```
  Download automático para Supabase Storage bucket `assets/images/{client_id}/`

- [x] **AC-5:** Rate limiting e cost tracking por chamada (adicionar em `llm_usage_log` ou tabela separada)

- [x] **AC-6:** Imagens de referência do Google Drive incluídas como input quando disponíveis (style reference/transfer)

- [x] **AC-7:** Testes: mock provider → gerar via mock → validar download para Storage

- Implemented `ImageGenerator` interface + `ImageOptions`/`ImageResult`/`StorageClient` types in `types.ts`
- `NanaBananaProvider`: POST to `/v1/generate` → poll every 2s max 30 attempts → download → Supabase Storage upload
- `KlingAIProvider`: Kuaishou task-based API — POST → `task_id` → poll every 3s → `task_result.images[0].url` → Supabase Storage
- `FallbackImageGenerator`: wraps primary+fallback, tries primary, on failure logs warn and delegates to fallback
